# Video Quality Lessons

This is the skill-local accumulation point for durable GJV1 Manim quality lessons. If an experiment produces guidance that should affect future videos, fold it into this file or into `SKILL.md` so the skill remains usable without outside notes.

## Stable Process

1. State the missing visual hypothesis before coding.
2. Budget the opening breath, mechanism beats, and final hold before rendering.
3. Build with minimal text, strict palette roles, and one dominant motion idea per beat.
4. Render the real video, then inspect proof frames that show the mechanism.
5. Run composition, crowding, or resting-mobject audits when framing or overlap is a risk.
6. Patch spacing, hierarchy, mechanism visibility, motion path, and cleanup in that order.
7. Keep only versions whose still frames look intentional without narration.

## Durable Lessons

- A mechanism must survive in a still frame. If the sampled proof frame does not show the cause of the motion, resample or strengthen the mechanism.
- The opening breath should show the initial structure first; do not spend it on a blank transparent frame.
- Final holds are part of the composition. Remove guides, slots, sleeves, shutters, or scaffolds once they stop causing the motion.
- In sequential negative-space transfers, retire each completed route scaffold as soon as its handoff lands. Keeping all route lines until the global cleanup can turn later proof frames into a residue field instead of quiet space.
- Use the default visual system first: white background, black text, gray structure, and primary red as the active accent. Reach for orange, yellow, green, blue, or purple only when the user asks for color or when extra categories need explicit separation.
- Square corners and straight rectangular edges are the default. Rounded forms should be an explicit style choice, not the baseline.
- Text should try `Open Sans` first, with Arial or the system sans-serif as fallback.
- Transparent clips with labels, diagram text, timeline cards, or captioned arrows need a local `page-background` stage so they remain inspectable outside a slide.
- Decorative transparent loops can stay transparent when the slide surface supplies contrast and the clip has no labels or diagram text.
- Device-frame or browser-frame embeds should not duplicate the surrounding Slidev frame inside the transparent Manim asset. Animate the content region only, and let the slide layout provide the chrome.
- For short transparent UI loops, keep receiver labels or result text hidden until the pulse or active route reaches the destination. Ghosted future text can make the handoff look like a static mockup instead of a mechanism.
- Step-reveal sequence loops need visible source states, receiver slots, and quiet route scaffolds in the opening breath. A single moving dot on an otherwise empty track reads as a placeholder, while prepared receivers make each handoff causal in still frames.
- Fade traveling pulses during cleanup before drawing a final outline, terminal arrow, or other resolved-frame device. Even when broad composition audits pass, a small pulse that lingers at the edge of a cleanup frame reads as disconnected residue in full-size review.
- When the preferred text font is not installed, choose an installed fallback before constructing `Text` mobjects. Repeated font fallback warnings make render logs noisy and can hide real Manim issues.
- Validate transparent VP9 WebM proof frames with the `libvpx-vp9` decoder, not only PyAV or the native FFmpeg decoder. Those paths can report `yuv420p` and opaque alpha even when the stream metadata has `alpha_mode=1` and libvpx exposes real alpha.
- Contact sheets are useful for broad passes, but suspicious thumbnails must be opened full size before patching.
- Camera-focus passes should hide or fully contextualize neighboring panels. Cropped fragments at the frame edge read as accidental unless they clearly remain context.
- For `ZoomedScene` image tours, inspect the built-in display pop-out before accepting it. The native pop-out can produce black inset frames before the zoom camera has a useful image; activating zooming without that animation and holding the live inset can read cleaner. Moving the zoom frame is also expensive because every frame rerenders the secondary camera, so use short movement beats with longer static holds. When the source image is meant to feel full-size but audits flag `low_visual_margin`, scale the image to leave a small review margin and scale the zoom target coordinates with it.
- For real-photo `ZoomedScene` tours, do not start from a synthetic placeholder when the request expects a real image. Prefer an official or clearly licensed source, store the local source asset in the spike, document provenance in the spike README, and generate a prepared 16:9 crop for the render. Choose zoom hotspots from actual pixel coordinates on the prepared crop, then inspect a contact sheet and any suspicious frames full size; plausible coordinates can land on empty or soft regions. Balance zoom-window width against source resolution so close-ups reveal real detail without turning the inset into blur or texture noise.
- Use `red guide tour` for large-diagram narratives where one primary-red companion marker carries the viewer through distant zones. The marker should be both wayfinder and actor: it can travel, pause, point, trigger, compress, orbit, split, or merge with local geometry, but each stop still needs its own readable mechanism.
- When a `ThreeDScene` surface is mixed with fixed-frame panels or labels, do not rely on a static filled `Surface` to remain visually stable through unrelated 2D beats. Keep the real 3D surface in an active proof beat, then settle to a fixed-frame 2D summary icon for later holds. Avoid no-op surface updaters as a visibility workaround; they can make renders stall or become impractically slow.
- When a spike reuses a fixed `media_dir`, clear the Manim staging directory before promoted review renders. Cached partial movies can preserve old wait frames after code changes and make contact sheets look unchanged even when the source was patched.
- Low edge clearance, off-center rest content, and low visual margins are composition problems even when no object is technically clipped.
- For imported SVGs, transform only compatible roles directly. Remove deleted roles first, establish the new primary body, then attach native Manim labels when text must stay readable.
- For generated diagram SVGs, keep inspectable source artifacts in the spike output, but animate stable role fragments and prefer native Manim connectors when imported arrowheads render too heavy.
- For direct Mermaid SVG insertion, keep the raw `mmdc` output for inspection, then use a Manim-compatible visible-geometry copy when `SVGMobject` would import hidden marker definitions or ignore SVG text. Recreate labels from the generated SVG coordinates, and parse `translate(x, y)` with comma-plus-space support so labels do not collapse onto one node.
- For Mermaid mindmap SVG unfolds, use edge ids such as `edge_0_1` as the parent-child hierarchy, not screen position. Mermaid mindmap labels are usually `foreignObject` HTML, so import the generated edge and node geometry as SVG fragments and rebuild labels natively from the SVG metadata. Fit rebuilt labels against the actual node body width, not the inner `foreignObject` width, and compose multi-word labels from word mobjects with a fixed gap so small branch labels do not collapse into odd tracking. Normalize the root `circle.basic label-container` as well as child `path.node-bkg` shapes; otherwise the root can keep Mermaid's fallback black fill while the branches use the intended palette.
- For all-type Mermaid galleries, render both SVG and PNG assets with Mermaid CLI. Use the Mermaid-rendered PNGs inside Manim when cross-diagram text fidelity matters, preserve the `.mmd` and `.svg` beside them for inspection, and put image-bearing cards in `Group` rather than `VGroup`. Keep page counters, final notes, and broad stage borders comfortably inside the frame because composition audits treat those support elements as real foreground.
- For generated diagram remaps, use specific receiver slots instead of one broad destination box, then remove those slots before any route pulse or resolved hold. Slot outlines that remain around docked nodes read as actor-to-outline crowding.
- In graph-style diagram videos, strict crowding audits can flag intentional connector-to-node proximity or pulse-on-route contact. Inspect overlays full size and treat the finding as blocking only when the contact hides an actor, collides with an outline after the causal beat, or muddies the mechanism.
- Slot-docking scenes need a held constrained proof frame with the dominant form inside an open receiver and the support forms outside the entrance. Hollow receivers can trigger strict crowding false positives because their bounding box surrounds the actor, so inspect overlays full size before deciding whether the slot is a real collision.
- Sequence handoffs need receiver cues before arrival, activation bars or route evidence during ownership, and a terminal artifact so the final hold has a center of interest.
- Long sequence routes need compact route-label chips; return paths should use a visibly different grammar from request paths.
- Bridge-span scenes should hold a proof frame where the dominant form is inside the passage while bridge rails remain visible above and below it. If source cleanup removes the left zone, recenter the target stage and resolved cluster before the final hold.
- Arc handoffs need one proof frame where the dominant form is visibly riding the curved path while supports stay calmer. If cleanup removes the source zone, recenter the resolved cluster so the final hold does not remain stranded in the old target lane.
- Table and formula scenes should identify source cells in place, compose the formula in a stable side zone, and transform the computed result into the destination cell.
- For API-gallery table scenes with a fixed top title, sparse one-table pages can read as top-heavy even when a footer is present. Move the main table and side legend block lower as one core composition instead of adding disconnected bottom text to satisfy centering.
- Continuation scenes should preserve the previous resolved state as a compact input, reveal destination scaffolds early enough to balance the frame, and populate output blocks progressively.
- Placeholder scaffolds should be removed or faded before text-bearing blocks appear. Do not morph placeholder panels directly into titles or list rows.
- When a left-to-right route later transforms into a centered system diagram, shorten or soften the ingress trail during the transformation beat. If the full route stays dominant while the new cluster appears, the eye keeps reading the old transfer lane instead of the new center of interest.
- Compression, squeeze, clamp, and throat-gate scenes need a visibly tighter constrained phase than both source and landing states.
- Deformation-flow scenes read cleaner when receiver slots are open bracket marks instead of closed outlines. Full circular or rectangular receivers can become actor-to-outline crowding after the actor lands, while bracket marks still reserve the destination during the opening breath.
- Retire each completed receiver mark immediately after its actor lands. Leaving closed slots, guide spines, or route rails behind settled actors can pass broad composition checks but fail full-size crowding review as scaffold residue.
- For source-to-target deformation layouts, validate centering at multiple beats. A layout can be centered during the opening breath and still drift when the first source actor moves into the gate or when the source rail becomes mostly empty.
- Clamp-close scenes should hold the pressure proof before the release, then remove abandoned source panels and clamp bars before morphing into the landing. If the clamp remains during the release, 0.3-second samples can read as support residue instead of resolved motion.
- Aperture and reveal scenes should not leave the destination stranded after the source zone disappears. Recenter or rebalance the destination stage during cleanup, and carry the active accent into the landing or remove it with the guide.
- Layered reveal scenes need more than a passive vertical divider. Give each lane a visible aperture or slot, compress the active layer into that aperture, and only then release it to the target so the reveal mechanism survives 0.3-second proof frames.
- Mask-transfer scenes should retire the source row, route lines, destination slots, mask band, and traveling accent before the compact landing morph. Faded source actors or an accent left inside the destination cluster can pass broad composition checks while failing strict actor-to-actor clearance.
- Orbit, fan-out, and fork scenes need enough spacing in proof frames to avoid reading as collisions or loose regrouping.
- Mind-map fan-outs should reveal branches in a vertically balanced order, such as top then bottom before filling middle lanes. Revealing multiple upper lanes first can make otherwise correct branch mechanics read as off-center for several seconds.
- Do not transform a seed or placeholder directly into a text-bearing mind-map box. Transform the box body first, then fade in the label so proof frames do not show warped intermediate text.
- For generated mind maps with category and child levels, reveal all category trunks before any category emits child leaves. Letting the first category populate its children while lower categories are still empty can make the proof frames read as top-heavy even when the final tree is balanced.
- A red-and-grayscale mind map stays readable when red is reserved for active generation routes and pulses, while settled categories, child boxes, slots, and guide residue move back into distinct gray values.
- For mind maps that should avoid visible connector lines, use `cellular sprout line`: staggered dot or cell chains along implicit curved paths. Fade the terminal bud out and fade the box body in instead of morphing a circle into a rectangle, and avoid outline focus flashes that can read as stray connectors.
- For plant-like mind-map connectors, use `organic fractal line`: a thick active main stem with thinner side tendrils and at most two recursion levels. Keep pending stems faint, reveal all category trunks before child branches, and fade settled routes to gray so organic detail does not compete with labels.
- For scale-like or thorn-like mind-map connectors, use `scale spine line`: repeated pointed polygons oriented by the tangent of an implicit route. Make active trunks large and dense enough to read as overlapping scales at full size, alternate side offsets so they do not become a dashed line, and inspect full-size proof frames because thumbnails can collapse the shape into dots.
- For aloe-like connector growth, keep `scale spine line` but switch from small diamond scales to wide lance-shaped polygons with several side offsets. The aloe reference is mainly about dense overlapping succulent leaf geometry; keep settled opacity low so the final hold does not become a heavy ribbon.
- Dense aloe-like spines need a darker narrow stroke on active leaves when the units visually merge. Lower the settled stroke opacity with the fill so the hold still reads as separate elements without turning into a conventional connector line.
- When testing non-diamond mind-map connector tokens, render candidates on the same route and keep a simultaneous final hold. Rectangles and triangles usually keep directional growth; squares and circles are clearer as separate elements but read more generic; stars are memorable but can become decorative noise.
- Snap recoil scenes need the destination slot or pressure surface visible before the snap, a held stretched overshoot proof frame, and early removal of target slots before support forms settle. Keep the wall far enough from supports to clear targeted crowding checks, then recenter the resolved stage after source cleanup.
- Edge-tension scenes read best when the destination edge exists as a temporary pressure surface before arrival, the overshoot is held against it in a proof frame, and abandoned source panels disappear before the final hold.
- Axis-drop cues should keep the causal drop vertical, hold it, remove it, and only then move the point along any later rail.
- Bumper deflect scenes read cleaner when the leader owns a held compression frame, supports move into separate release lanes before the final morph, and passive destination slots disappear before the landing.
- Cradle catch scenes need a held proof frame where lower support pads visibly receive the dominant form. After that proof, separate support dots downward and outward and remove cradle/backing scaffolds so the resolved hold keeps clear actor-to-actor spacing.
- Magnet capture scenes should show a prepared receiver during the opening breath, hold the leader compressed inside the pocket before supports move, retire target slots and field rails immediately after their causal beat, and fade the receiver/core before the landing morph. If a bracket receiver is audited strictly, separated strokes or clear corner gaps prevent one U-shaped guide from becoming a false overlap box.
- When a reviewer names a timestamp, validate that exact timestamp and the surrounding transition frames in the promoted video.
- Poster images must be updated when the terminal video state changes.
- Promote rendered videos and posters by latest modified time, not lexicographic path order. Manim can leave reused quality folders such as `900p30` and `900p15` beside each other, and sorted path order can copy a stale render.
- When a spike reuses one fixed Manim staging directory for several transparent loop variants, clear that staging directory before a review render so cached partial movies and stale promoted outputs cannot contaminate the proof sheet.
- When target slots or receiver outlines must stay hollow, set stroke opacity directly instead of applying group opacity. Group opacity can turn stroke-only placeholders into filled disks and make the opening breath look like unresolved actors.
- Do not animate `set_opacity()` on a group that contains `always_redraw`, `TracedPath`, or other dynamic mobjects. The child family can change between frames and raise interpolation errors; reveal those systems with a preview-to-real handoff, `FadeIn` or `FadeOut`, or explicit opacity changes on stable static children.
- Decorative backing strips and review frames still count as rest-state composition objects. If a resting audit flags low edge clearance on neutral structure, shrink the structure or widen the camera before accepting the hold.
- Once a scaffold or backing shape disappears, it should not reappear for a late transition beat. Fade the support once and let the landing continue without that object.
- Avoid reusing a transient pulse across separated handoffs if it must disappear between them. A route that draws itself once usually reads cleaner than a dot that vanishes, reappears, and vanishes again.
- Text-bearing terminal cores should use perimeter halos or behind-core rings for pulses. A filled accent crossing the glyph can make the proof frame unreadable even when the pulse is visually centered and intentional.
- For `GrowArrow` teaching scenes, treat the animation as an `Arrow` or `DoubleArrow` introduction from the declared start point. If the lesson needs a curved route, use `Create(CurvedArrow(...))` unless the specific curved object has already been rendered successfully with `GrowArrow`; non-`Arrow` tipable routes can stall or become brittle.
- When code-like labels are part of the visual lesson, use a monospaced font for those labels. Underscores and punctuation can become too faint or visually ambiguous in proportional diagram text.
- For `Broadcast` teaching scenes, treat `focal_point` as the visual center of the emitted copies; the passed mobject supplies size and style, not a preserved destination position. Stroke-only mobjects read as clean shells, while any nonzero fill opacity makes the broadcast fade filled sheets. Use `remover=False` only when the final residue is the point of the lesson, because expanded copies remain in the scene and need their own composition budget.
- For non-default aspect-ratio renders, make resting-mobject audits use the same frame shape as the real render. Add a spike-local setup function that sets `config.pixel_width`, `config.pixel_height`, and `config.frame_height = config.frame_width * pixel_height / pixel_width`, then pass it with `--setup-call`; otherwise portrait variants can produce false outside-frame findings.
- Run dry-run resting audits for multiple variants sequentially or give each audit a separate `--media-dir`. Parallel Manim text rendering can share cache/staging paths and produce transient empty-SVG parse failures that are unrelated to the composition.
