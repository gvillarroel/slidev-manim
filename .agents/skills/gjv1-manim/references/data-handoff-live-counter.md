# Data Handoff To Live Counter

Use this pattern when a table, list, or source row needs to express a transformation into a derived numeric result. It is a promising direction for narration because the viewer can see source data become a changing value instead of only seeing the final number.

## Narrative Shape

1. Cue the source row or source cells first.
2. Move or reveal the selected source values into a separate formula or aggregation zone.
3. Produce one transient result token from that zone.
4. Start the live counter only while that result token is visibly in flight.
5. Land the resolved value into the destination cell, chart, or summary slot.
6. Remove temporary supports and hold a clean final state.

## Why It Works

- The transformation has a visible cause, transit phase, and destination.
- The counter reads as evidence of the handoff, not as decoration.
- The side formula zone protects dense tables from overlays and duplicate text.
- The final destination value can inherit meaning from the motion that produced it.

## Polish Targets

- Keep source values readable while they are being selected; avoid markers directly on top of numbers.
- Align the side mechanism to an explicit column or grid so it does not feel like a floating annotation.
- Sample the formula-ready frame, counter-in-flight frame, result-landing frame, and final no-support hold.
- Fade source-side terms before the result token crosses back to the destination.
- Recenter or rebalance the final hold after the side mechanism disappears.
- Prefer a single counter and one active handoff token per beat; extra animated terms can make the transformation read as bookkeeping.

## Good Uses

- Table row aggregation into a subtotal or score.
- Classification confidence becoming a visible numeric counter.
- Before-and-after metric changes where the result needs to feel computed.
- Slide narration moments where the spoken line says what changed while the motion shows how it changed.
