# Browser-Native Weave Lessons

- In browser-native weave or gate narratives, do not let the resolved hold keep the same tight enclosure as the pressure beat. If the top, bottom, and side bands stay collapsed around the actor, the ending reads like a cage instead of a woven resolution.
- For the final hold, pull the settled bands outward enough to reveal a calmer central void and demote the active travel path below the settled structure. The mechanism can stay visible as a memory trace, but it should no longer dominate the resolved frame.
- If the transformation uses a diagonal or curved travel trace to prove the weave, fade that trace almost completely by the terminal hold. A visible route memory across the final dot can make the ending read as a sampled transition frame instead of a resolved emblem.
- Prefer separated corner marks over one closed terminal outline when the weave resolves around a compact central actor. Corner marks preserve orthogonal structure on desktop and mobile proof frames without enclosing the actor in one heavy box.
